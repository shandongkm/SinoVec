# SinoVec 开发路线图

本文档记录了 SinoVec 的过去、现在和未来计划。

## ✅ 已完成 (v1.0.x)

### v1.0.0 (2026-04-09)
- 初始发布
- 向量 + BM25 混合检索（动态权重）
- LLM 查询扩展（可选，通过环境变量控制）
- LLM 重排（可选，候选数超过阈值时触发）
- 时间衰减（半衰期 30 天，可配置）
- HTTP API (`/search`, `/health`)
- CLI 命令 (`add`, `search`, `stats`, `list`, `delete`, `summarize`)
- 自动记忆提取 (`extract_memories.py`)
- 会话历史索引 (`session_indexer.py`)
- 记忆热度晋升 (HOT/WARM/COLD)
- 语义+时效去重 (`cmd_dedup`, `cmd_dedup_deep`)
- 血缘追踪 (`memory_lineage` 表)
- Docker 一键部署
- systemd 服务支持

### v1.0.1 (2026-04-15)
- **修复**：`fts` 生成列与手动插入冲突（PostgreSQL 报错）
- **修复**：`extract_memories.py` / `session_indexer.py` 中 DB_CONFIG 不一致（端口 5433，用户 openclaw）
- **修复**：移除 `generate_vector` 中硬编码代理 `127.0.0.1:7890`
- **修复**：`cmd_dedup` 中硬编码常量替换为环境变量
- **修复**：`_ollama_generate` 异常时返回空字符串
- **新增**：`/stats` API 端点（返回记忆总数、召回次数等）
- **新增**：辅助脚本支持 `--dry-run` 模式
- **新增**：辅助脚本全局模型缓存（避免重复加载 FastEmbed）
- **优化**：`install.sh` 支持虚拟环境、备份配置、动态生成 systemd 服务
- **优化**：统一数据库默认值为 5433/openclaw

## 🔄 计划中 (v1.1.0)

### 高优先级

#### 1. 性能基准测试
- **目标**：提供官方性能数据（不同数据量下的 QPS、延迟、召回率）
- **方法**：编写 `benchmark.py`，测试 1k/10k/100k 记忆时的查询性能
- **交付**：性能报告（Markdown + 图表），集成到 CI

#### 2. Web 管理界面
- **目标**：提供可视化界面，方便非技术用户管理记忆
- **技术栈**：Flask + 原生 HTML/CSS（轻量）
- **功能**：记忆列表、搜索、编辑、删除、统计图表
- **交付**：`web_ui.py` 独立服务，可单独启动

### 中优先级 (v1.2.0 或社区贡献)

#### 3. 多模态支持（图片记忆）
- **目标**：支持图片记忆，实现图文混合检索
- **技术挑战**：CLIP 模型输出维度（512/768）与现有 512 维向量对齐
- **方案探索**：使用 512 维 CLIP 模型或升级 pgvector 支持混合维度
- **交付**：`cmd_add --image` 参数，检索时自动匹配图片

### 低优先级 (未来版本)

#### 4. Qdrant / Milvus 后端支持
- **目标**：支持专用向量数据库，满足大规模分布式部署需求
- **方法**：抽象向量存储接口，实现适配器
- **交付**：迁移指南、配置示例
